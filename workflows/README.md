# Workflows

Markdown SOPs defining objectives, required inputs, tools to use, and expected outputs.

## Structure of a Workflow

Each workflow file should include:
- **Objective** — what this workflow accomplishes
- **Required Inputs** — what's needed before starting
- **Steps** — which tools to run and in what order
- **Expected Outputs** — what success looks like
- **Edge Cases** — known failure modes and how to handle them
